## 4. 必须保持不变的公共接口

本单元引入的公共接口**只有一处**，且 G1 起所有依赖 U01 的代码必须遵守：

```python
# reacquisition_first_ring_core/time_point.py

# 内部表示：有符号 64 位纳秒整数。
# 取值范围：[-2^63, 2^63 - 1] 纳秒，约 ±292.47 年。
TimePoint = int  # 简化表示；类型层面只是 int，运行时由 helper 校验

# 哨兵值：
TIME_POINT_INVALID: int = 2**63 - 1   # 显式无效（区分"未初始化"）
TIME_POINT_MIN: int = -(2**63)        # 可表示最小值
TIME_POINT_MAX: int = 2**63 - 2       # 可表示最大值（保留 INVALID 为哨兵）

# 校验函数签名（必须保持）：
def is_valid(tp: int) -> bool: ...
def make_valid(nanoseconds: int) -> int: ...   # 越界或 INVALID 时抛 ValueError
def nanoseconds_since_epoch(tp: int) -> int: ...  # 等同 tp，但语义命名

# 异常类（必须保持）：
class TimePointError(ValueError): ...
```

**冻结约束**：

- `TimePoint` 类型别名不得改为 `class`、NamedTuple、dataclass 等任何包装类型。后续 G1+ 可基于此扩展，但**不得破坏**此别名。
- `TIME_POINT_INVALID` / `TIME_POINT_MIN` / `TIME_POINT_MAX` 常量值与名称不得变更。
- `is_valid` / `make_valid` / `nanoseconds_since_epoch` 函数名与签名不得变更。
- 探针脚本产出的 CSV 列顺序与 JSON 字段名见第 11 节，**不得变更**。

## 5. 输入与输出数据结构

### 5.1 `TimePoint` 内部表示

| 项 | 值 |
|---|---|
| 类型 | Python `int`（有符号 64 位纳秒） |
| 单位 | 纳秒 |
| 范围 | `[-(2^63), 2^63 - 1]` 纳秒 |
| 哨兵 | `2^63 - 1` 表示 `INVALID` |
| 坐标系 | 无（仅时间维度） |
| 默认值 | `TIME_POINT_INVALID`（避免零值隐式表达"有效 0 纳秒"） |

### 5.2 探针输入数据源

本单元**只支持模拟输入**（环境阻塞，真实 PX4 模式留待目标 ROS 2 环境）：

```python
@dataclass(frozen=True)
class SimulatedPx4Sample:
    monotonic_ns: int              # 单调稳态时钟，单位 ns
    ros_now_ns: int                # ROS 当前时间模拟值（模拟 /clock）
    px4_timestamp_sample_ns: int   # PX4 timestamp_sample 模拟值（μs → ns）
    px4_timestamp_ns: int          # PX4 timestamp 模拟值（μs → ns）
    simulated_offset_ns: int       # 注入的固定偏移，用于测试
    simulated_drift_ns: int        # 注入的漂移，用于测试
```

模拟样本由 `tools/probe_px4_time.py` 内置的 `_simulate_*` 工厂函数生成，覆盖：固定偏移、漂移、时间倒退、重复时间戳、`/clock` 重置、数值溢出 6 类场景。

### 5.3 探针输出

#### CSV 列（顺序固定）

```text
monotonic_ns,ros_now_ns,px4_timestamp_sample_ns,px4_timestamp_ns,delta_offset_ns,delta_drift_ns,clock_reset_flag
```

#### JSON 摘要字段（字段名固定）

```json
{
  "mode": "simulated",
  "sample_count": 0,
  "duration_ns": 0,
  "offset": {
    "mean_ns": 0.0,
    "std_ns": 0.0,
    "p95_ns": 0.0,
    "max_ns": 0.0
  },
  "drift": {
    "slope_ns_per_s": 0.0,
    "intercept_ns": 0.0
  },
  "reversal_count": 0,
  "duplicate_count": 0,
  "clock_reset_count": 0,
  "overflow_count": 0,
  "input_files": [],
  "output_csv": "",
  "output_json": ""
}
```

> 输出文件路径由命令行参数 `--output-dir` 指定。**不得写入源码包内部**，仅写入运行时指定目录。

## 7. 错误码和异常行为

| 场景 | 行为 |
|---|---|
| `make_valid` 输入越界 | 抛 `TimePointError("TimePoint out of range: ...")` |
| 探针脚本 `--mode live` 在 G0 调用 | 抛 `NotImplementedError("live mode requires ROS 2 + rclpy + px4_msgs; environment blocked on Windows dev machine")` |
| 探针脚本 `--output-dir` 不存在 | 自动创建；创建失败抛 `OSError` |
| 探针样本字段出现 ≥2^63 整数 | 计入 `overflow_count`，**不抛异常**，记录到 CSV 行的 overflow 标志列 |
| 探针运行时 Ctrl+C | 优雅退出，已写入的 CSV 行保留，JSON 摘要不写 |
| pytest 收集到任何包导入失败 | 整个测试套件返回非零，不静默 |

**启动失败条件**：包骨架 import 失败、`pytest` 不能发现任何测试、`probe_px4_time.py --mode simulated` 在固定偏移场景下崩溃。这三项任一发生即视为 U01 不通过。
